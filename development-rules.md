開発時の運用ルール
----------------------------------

common
==================================
 * issueやPRにコメントする際は先頭に必ず誰に対するコメントなのかを`@アカウント名`で明記すること
 * コメントした際はIRCでその旨該当者に連絡すること
 * 自分で作成したPRを自分でマージしないこと

developer
==================================
 * masterブランチに直接pushしてはいけない
 * issueに対応するブランチを作成し、PRを作成しレビュー依頼をする
 * コミットは意味ある単位で実施すること
   * 無暗にコミットをしまくるとレビューしづらいしコミット履歴が汚くなる
 * ビルドの通らないものは原則NG
 * conflictが発生してよくわからなくなったら、その作業ブランチを捨てて、masterから再度ブランチを再作成し作業すること
   * よくわからないままrebase -> force pushして資材を壊す・汚すを避けたい
 * レビューコメントに対して資材を修正した際は、指摘コメントに返信する形で元コメントを編集し、どのコミットで対応したのかを明記すること

reviewer
==================================
 * PRに対してレビューを実施する
 * レビューコメントはPRに対してコメントする
   * commitやdiffにコメントするとrebase -> force pushした際に消えるので極力しない方針で
 * レビューの指摘事項は該当行のリンクをコメントに記載すること
   * [URL]#Lnn
 * レビューOKの場合、ビルドが通っているか確認すること（GitHub Pull Request Builderによるjenkinsの自動ビルドを想定）
